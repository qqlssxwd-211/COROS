# 运动分析与建议 设计规范

> 最后更新：2026-05-22

**目标**：基于 COROS 活动数据，新增「运动分析」Tab，提供能力评估、训练负荷分析和个性化运动建议。

**数据基础**：`/activity/query` 返回的全部活动数据（40+ 字段），无睡眠/HRV/VO2max 等额外接口。

**数据门槛**：≥5 条活动可展示估算值，标注「参考值」；≥30 条标注「较可靠」。

---

## 一、功能模块

### 1. 能力评估 (AbilityPanel)

**1.1 VO2max 估算**
- 算法：从历史最佳跑步成绩反推 VDOT（Jack Daniels 公式），再换算 VO2max
- 公式：`VDOT = (-4.6 + 0.182258 * velocity + 0.000104 * velocity^2)` 其中 velocity 为 m/min
- 数据充足（≥30 条、含不同距离跑步记录）：取多次估算中位数
- 数据少：标注「参考值，建议积累更多数据」

**1.2 成绩预测表**
- 从当前 VDOT 推算各距离完赛时间：1.6K / 5K / 10K / 半马 / 全马
- 使用 Jack Daniels 预测公式，展示 5 个距离的预估完赛时间和等效配速
- 以表格卡片展示

**1.3 配速-心率效率**
- 效率指数 = `速度(m/s) / 心率 * 100`
- 折线图趋势：上升表示同样心率跑得更快，体能提升
- 标注历史最佳效率和近 4 周变化率

**1.4 历史最佳趋势**
- 提取每月最远距离、最快 5K/10K、最大爬升
- 按月展示变化，柱状图

### 2. 训练负荷分析 (LoadPanel)

**2.1 急慢性负荷比 (ACWR)**
- 急性负荷 = 最近 7 天 trainingLoad 总和
- 慢性负荷 = 最近 28 天平均训练负荷
- ACWR = 急性 / 慢性
- 折线图展示，标注区间：
  - `< 0.8` 灰色（训练不足）
  - `0.8 - 1.3` 绿色（合理）
  - `1.3 - 1.5` 黄色（偏高）
  - `> 1.5` 红色（过度训练风险）

**2.2 训练强度分布**
- 跑步活动按配速分级：轻松跑 / 节奏跑 / 间歇跑 / 冲刺
- 分级基于最佳配速百分比
- 饼图展示占比，对比推荐 80/20 比例

**2.3 周度训练负荷趋势**
- 柱状图：每周训练负荷 + 7日均线
- 叠加折线：慢性负荷（28日均线）

**2.4 恢复状态**
- 基于最近 3 天活动负荷 × 衰减系数（第1天×0.7, 第2天×0.3, 第3天×0.1）
- 进度条样式：已恢复 / 基本恢复 / 需要休息

### 3. 建议与规划 (SuggestionPanel)

**3.1 周度训练建议**
- 取近 4 周平均训练量，推荐下周目标：维持或增加 ≤10%
- 强度配比：轻松 80% : 强度 20%
- 建议运动频次（基于近 4 周平均频次）
- ACWR 偏高时自动建议减量周

**3.2 状态提示卡片**
- 综合配速心率效率 + ACWR + 训练频率，一句话状态描述
- 使用醒目卡片，情绪化提示

**3.3 目标设定**
- 输入目标类型和数值（月跑量、10K成绩等）
- 自动计算每周训练量建议
- 进度追踪：本月完成百分比

**3.4 交叉训练建议**
- 某类运动占比 >90% 时提示多样化
- 推荐搭配方案

## 二、组件结构

```
src/
├── components/tabs/AnalysisTab.tsx      # 主容器，TabShell + 三段滚动布局
│   └── components/analysis/
│       ├── AbilityPanel.tsx             # 能力评估卡片组
│       ├── LoadPanel.tsx                # 训练负荷分析
│       └── SuggestionPanel.tsx          # 建议与规划
└── lib/analysis.ts                      # 纯计算函数（无UI依赖）
```

**AnalysisTab** 使用 TabShell，内部为移动端友好的纵向滚动布局，三个面板依次排列，每个面板带标题和卡片容器。

**analysis.ts** 导出纯函数：
- `estimateVO2max(activities)` → `{ vo2max, confidence }`
- `predictRaceTimes(vo2max)` → `Record<distance, { time, pace }>`
- `calcEfficiencyIndex(runningActivities)` → `DataPoint[]`
- `calcACWR(dailyLoads)` → `{ acute, chronic, ratio, status }`
- `classifyIntensityZones(activities, bestPace)` → `Record<zone, { count, percent }>`
- `calcRecoveryStatus(recentActivities)` → `{ score, label }`
- `generateWeeklySuggestion(activities, acwr)` → `{ distance, intensity, frequency }`

---

## 三、数据流

```
activities (DataContext)
    │
    ├──→ analysis.ts (计算)
    │       ├── VO2max → 成绩预测
    │       ├── 效率指数趋势
    │       ├── ACWR → 负荷状态
    │       ├── 强度分布
    │       └── 恢复状态
    │
    └──→ AnalysisTab (展示)
            ├── AbilityPanel
            ├── LoadPanel
            └── SuggestionPanel
```

## 四、UI 约定

- 延续现有暗黑主题（`bg-[rgba(5,5,5,0.97)]`、`border-white/6`）
- 卡片使用 `rounded-2xl border border-white/5 bg-white/[0.02]`
- 图表使用 echarts-for-react，延续现有 LineChart/BarChart 模式
- MetricCard 复用现有组件
- 移动端：单列布局，面板间用分割线分隔
- 数据不足时显示空状态占位 + 提示文案
- 估算值旁用 Badge 标注「参考值」或「较可靠」
